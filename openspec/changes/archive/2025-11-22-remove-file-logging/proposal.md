# Change: 移除文件日志功能

## Why

当前扩展将日志同时输出到三个位置：文件、OutputChannel 和 Console。文件日志存在以下问题：

1. **维护成本高**：需要管理日志文件的创建、清理、大小限制等
2. **用户价值低**：大多数用户只需要 OutputChannel 即可查看日志
3. **资源浪费**：每次激活都创建新日志文件，占用磁盘空间
4. **管理复杂**：缺乏自动清理机制，旧日志文件会持续累积

移除文件日志可以简化实现，减少维护负担，同时保留核心的日志查看功能（OutputChannel）。

## What Changes

- 移除日志文件的创建和写入逻辑
- 移除日志文件路径相关的配置和状态
- 移除 "显示日志文件" 和 "打开日志目录" 命令
- 保留 OutputChannel 和 Console 日志输出
- 保留日志级别配置功能

## Impact

- **受影响的 specs**: `logging` (新建)
- **受影响的代码**:
  - `src/utils/logger.ts` - 移除文件日志相关代码
  - `src/extension.ts` - 移除日志文件相关命令注册
  - `package.json` - 移除日志文件相关的命令声明
- **用户体验**: 
  - 用户将无法通过文件查看历史日志
  - 需要通过 OutputChannel 查看日志（功能保留）
  - 移除了 "显示日志文件" 和 "打开日志目录" 命令
- **破坏性变更**: 是（移除了现有功能）
